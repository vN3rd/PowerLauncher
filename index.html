PowerLauncher

Customizable PowerShell WinForms GUI launch-pad

View project onGitHub

About

A fully customization PowerShell GUI launch-pad. The original idea came from a similar GUI I developed for a family member in order to make it easy for them to open the apps that the commonly use. As part of that process, I started to expand the idea to something that catered more to a 'power-user' workflow; which really translated to something I could use to quickly modify the personality of a launch-pad type interface.

The interface can be re-sized and the buttons will automatically change orientation for a bit more of a custom layout. The only requirement is that the config file and binary exist in the same directory. Being that it is portable, experiment with creating separate directories for different use cases; ex: a version for App Installations, a version for shortcuts/common apps, maybe a version that executes some report scripts, etc...

How It Works

Upon launch, the application looks for the config file and if it exists, it loads all of the variables and functions in. There are some import and mappings that occur in the Global.ps1 file that pull in and assign what's defined in the config file, to 'application' variables. When a button is pressed, it calls the function defined for that button in the config file. Thus, if you make changes to the variable names in the config file, things will almost certainly break.

Getting Started

  1. Download the source code or the binary (.exe) file that matches the architecture type (32/64-bit)
    • 64-bit Binary Download
    • 32-bit Binary Download
  2. Copy the contents of the blank sample config file and paste them into your own PowerLauncher_Config.ps1 file.
  3. Place the .exe and the config file in the same directory and launch

Troubleshooting

  • If the app launches, and despite having a config file present, no buttons load, take a second pass at your config file and make sure there are no syntactical errors. Also, make sure you are running the right version (32/64-bit)
